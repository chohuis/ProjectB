## 2) M1/M2 기준 상태

### M1 (고교-대학 전환 구간)
- 완료/반영
  - W50 진로 선택 트리거
  - W51 드래프트 트리거
  - W52 폴백 선택 트리거
  - 폴백 지원(대학 최대 3, 독립 최대 3, 체육부대 지원) 결과 저장
- 남은 보강
  - 대학/독립 합격 확률표 외부 데이터화
  - 실패/합격 결과 메시지 템플릿 확장
  - 분기별 QA 체크리스트 자동화

### M2 (시즌 종료 안정화)
- 완료/반영
  - 시즌 종료 모달에서 고교 시즌 종료 동기화 호출
  - 오프시즌 NPC 라이프사이클 정규화(은퇴/로스터 정리)
  - 라이프사이클 결과 로그/뉴스 메시지 기록
- 남은 보강
  - 리그별 로스터 최소/최대 규칙 데이터 외부화
  - 라이프사이클 리포트 UI 가독성 개선
  - 다년 시뮬레이션 안정성 점검(장기 누적 테스트)

## 3) 현재 확정된 핵심 규칙
- 고교 3학년:
  - W50 진로 의향 선택
  - W51 드래프트
  - W52 드래프트 미지명 시 폴백 선택
- 폴백:
  - 대학 3개, 독립 3개, 체육부대 지원 결과를 보고 최종 선택
  - 모두 실패 시 일반 입대 선택 가능

## 4) 리스크/주의사항
- 한글 인코딩 깨짐 이력:
  - Svelte 파일/템플릿 편집 시 UTF-8 유지 필수
- 매니페스트/인덱스 동기화:
  - `resource/data/master/_manifest.json`
  - `resource/data/master/entities/players/_index.json`
  - 데이터 갱신 스크립트 실행 시 자동 변경될 수 있음
- 점검 미완:
  - 현재는 기능 구현 우선, 통합 회귀 점검은 별도 라운드 필요

## 5) 다음 우선순위(권장)
1. M1/M2 통합 점검 라운드 실행(고교 1시즌~진로확정~시즌전환)

3. 라이프사이클 결과를 연감/뉴스로 구조화(검색 가능)
4. 프로 구간(M3) 상세 설계 착수(2군, 강등 보호, 은퇴 규칙)
5. 이벤트 메시지 템플릿 확장 및 중복 표현 정리
