## 1. 스타일 기준 이미지 — 반드시 확인

### `apps/ui/src/shared/assets/match-demo/stadium-scene-concept.png`

이 이미지가 **모든 품질의 기준**입니다.

이미지 안에 이미 완성된 캐릭터가 세 명 있습니다:

**① 투수 (화면 왼쪽 하단)**
- 뒷모습 (3/4 후면), 흰 유니폼 + 네이비 캡·소매
- 왼무릎 들어올린 릴리스 직전 포즈
- 치비 비율 (머리 크고 몸 짧음)
- 픽셀 단위의 선명한 외곽선, 부드러운 명암

**② 포수 (홈플레이트 뒤)**
- 뒷모습, 어두운 회색/검정 풀 장비 (마스크·가슴보호대·정강이)
- 쪼그려 앉아 사인 자세

**③ 타자 (오른쪽, 측면)**
- 오른쪽 면이 보이는 측면 시점
- 빨간 헬멧 + 빨강/흰 유니폼
- 배트 어깨에 올리고 대기 중

### 이 캐릭터들의 아트 스타일을 그대로 따라야 합니다:
- AI 이미지 생성 품질의 픽셀아트
- 치비 비율 (머리 높이 ≈ 몸 높이의 50~60%)
- 각 색상에 하이라이트/그림자 1~2단계
- 명확한 다크 아웃라인
- 의상 디테일 (주름, 번호, 장비 디테일)

> ❌ SVG `<rect>` 더미 픽셀아트 수준은 거부
> ✅ `stadium-scene-concept.png` 캐릭터 품질이어야 함

## 3. 팀 컬러 — Placeholder 색상 시스템

PNG는 런타임에 팀 고유 색상으로 교체됩니다.  
**아래 정확한 hex 값으로 유니폼을 채색해야 합니다.**

### 투수 스프라이트
| 영역 | Placeholder 색상 | 설명 |
|---|---|---|
| 유니폼 주색 (몸통·바지·헬멧 메인) | `#1E3A8A` | 팀 주색으로 교체될 영역 |
| 유니폼 보조색 (줄무늬·번호) | `#FFFFFF` | 대부분 팀이 흰색 보조색 사용 |
| 피부 | `#FDBCB4` | 고정 |
| 글러브 | `#8B5E3C` | 고정 |
| 스파이크·벨트 | `#1A1A1A` | 고정 |
| 외곽선 | `#0A0A0F` | 고정 |

### 타자 스프라이트
| 영역 | Placeholder 색상 | 설명 |
|---|---|---|
| 유니폼 주색 (헬멧·몸통·바지 메인) | `#8B1A1A` | 상대 팀 주색으로 교체될 영역 |
| 유니폼 보조색 | `#FFFFFF` | 고정 |
| 피부 | `#FDBCB4` | 고정 |
| 배트 | `#6B3A1F` | 고정 |
| 스파이크·벨트 | `#1A1A1A` | 고정 |
| 외곽선 | `#0A0A0F` | 고정 |

### 포수·심판 스프라이트
| 영역 | 색상 | 설명 |
|---|---|---|
| 전체 장비·유니폼 | `#2A2A2E` (포수), `#1C1C20` (심판) | 고정, 팀 컬러 교체 없음 |
| 피부 | `#FDBCB4` | 고정 |

> **중요**: placeholder 색상(`#1E3A8A`, `#8B1A1A`)이 정확히 일치해야 Canvas API로 픽셀 교체가 가능합니다.  
> 명암 처리 시 이 색상의 밝은 버전(`lighter`, `+30%`)과 어두운 버전(`darker`, `-30%`)은 허용됩니다.  
> 런타임에 세 가지 톤(밝음·기본·어두움) 전부 팀 컬러 기반으로 자동 생성됩니다.

## 6. 납품 후 작업 (개발 측)

PNG 납품 시 Canvas API로 런타임 팀 컬러 교체:

```
투수 스프라이트: #1E3A8A → myTeam.colors[0]
타자 스프라이트: #8B1A1A → oppTeam.colors[0]
```

팀이 60개, 주색이 24가지이므로 PNG 파일은 1세트(14개)만 제작하면 됩니다.
