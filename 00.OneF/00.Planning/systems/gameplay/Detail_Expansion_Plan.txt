-10) 텍스트 프로토타입 (WPF)
- 메인 UI 구조: 상단 프로필/투구 정보, 좌측 섹션 리스트(홈/상태/학업/메시지/로스터/일정/훈련/재정/기록/팀), 우측 패널에서 섹션별 콘텐츠 스위칭.
- 홈 화면: 시즌 요약, 최근 5경기 카드, 다가오는 이벤트 리스트를 배치하고 MatchOverlay 외부에서는 요약만 보여 줌.
- 내 정보 화면: FM 스타일 프로필 카드 + 신체·기술·멘탈 3열 스탯 패널(범위 비표시).
- 로스터 화면: 감독·코치 + 선수 리스트(더블클릭 시 팝업)와 세부 스탯 그리드.
- 좌측 메인 메뉴: ListBox + DataTemplate 카드 스타일, 선택 시 배경 하이라이트 유지.
- 주간 스케줄 카드: 7일 × Morning/Afternoon/Evening 요약 텍스트, 선택 시 편집 가능한 형태.
- 훈련 세션: 콤보로 선택한 Morning/점심/저녁 조합을 세션으로 저장/삭제하는 리스트 제공 → 추후 주간 계획 자동 배치에 연결.
- 훈련 패널: 콤보/텍스트 영역/세션 리스트/주차별 계획을 한 화면에 수납하도록 폭·높이를 조정하고, 버튼은 하단 우측 정렬.
- 주차별 훈련 계획: 저장된 세션을 주차 단위로 배치하고, 현재 주차에는 자동으로 행동이 반영되도록 구현.
- 전체 일정: Stage_Rules 기반 이벤트를 월/주 단위로 나열하는 리스트 뷰 추가(입학·학업·경기·합숙 등 카테고리 표시).
- 월별 캘린더: 6x7 캘린더 셀에 날짜/주요 이벤트 요약을 출력, ComboBox로 월 전환.
- 학교 선택 오버레이: 8개 학교 카드에 평균 OVR/강점/주요 선수/로스터 규모/루프 프리뷰를 표시하고, 우측 패널에서 철학·운영 지표·주요 선수·1학년 루프 타임라인을 확인.
- 테스트 편의를 위해 시작 학년(1/2/3)을 ComboBox로 선택 → 해당 학년 루프가 즉시 로드되어 PlannedEvents 검증 가능.
- 홈/상태 섹션: 현재 시즌 데이터와 동일한 루프 타임라인(다가올 이벤트 리스트)을 공유 데이터 소스로 연결해, 홈 위젯과 상태 화면에서 실시간으로 확인 가능.
- 경기 시뮬레이터: 경기 이벤트 발생 시에만 투구 버튼/구종 선택이 활성화되도록 상태 표시.
- 경기 로그는 MatchOverlay 안의 TextBox/결과 패널에서만 누적해 스크롤 처리, 메인 홈에서는 요약만 노출.
- 기록/로그: 전용 `기록` 섹션에서 `_game.LogEntries`를 일자별 Expander + 항목 리스트로 표시, 하루 진행 즉시 팝업(DailyLogOverlay)으로 해당 로그를 보여 준 뒤 기록 섹션과 동기화.
- 구현 메모: 좌측 ListBox + DataTemplate 카드, SelectionChanged로 콘텐츠 스위칭. 추후 Godot UI 설계 시 참고.
- 루틴 시스템 v1(2026-03-05): 루틴 카드 3종(밸런스/출전 대비/멘탈·학업) + 주간 캘린더 + 숙련도 링 + 감독 추천 버튼 + 일일 선택 카드(멘토/라이벌/대회 전략)를 프로토타입에 적용했다. 주말·대회일 잠금 처리, 루틴 숙련도 증감, 선택 카드 결과 로그 반영까지 연동 완료. 루틴 카드에는 편집/삭제/복원 액션과 커스텀 루틴 추가/이름 변경/행동 조합 편집을 지원하는 오버레이가 포함돼 세부 훈련 구성을 직접 제어할 수 있다.
