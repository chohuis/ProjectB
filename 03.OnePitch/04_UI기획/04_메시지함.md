## 1. 단일 채널 3종 흡수

| 종류 | 형태 | 근거 |
|---|---|---|
| **이벤트** | 본문 + 선택지(그 자리에서 [02_이벤트](../02_기획/콘텐츠/02_이벤트.md) §3 효과 반영). 단, 협상·트레이드 등 다단계는 전용 화면으로 위임([03_메시지_알림](../02_기획/콘텐츠/03_메시지_알림.md) §2) | [02_이벤트](../02_기획/콘텐츠/02_이벤트.md) §6 |
| **알림** | 본문만, 선택지 없음(확인만) | [03_메시지_알림](../02_기획/콘텐츠/03_메시지_알림.md) §1 |
| **뉴스** | 비긴급, 최신순 하단 | [05_뉴스_미디어](../02_기획/콘텐츠/05_뉴스_미디어.md) §4 |

## 2. 정렬 · 긴급도

- **정렬**: 긴급(블로킹) 최상단 고정 → 나머지 최신순([03_메시지_알림](../02_기획/콘텐츠/03_메시지_알림.md) §4).
- **블로킹 표시**: 긴급 이벤트는 응답 전까지 진행 차단([02_이벤트](../02_기획/콘텐츠/02_이벤트.md) §6) — Continue가 자동으로 이 항목을 최상단에 열어줌([00_개요](00_개요.md) 결정 2).
- **뱃지**: 비긴급 미확인 건수 표시(진행은 계속).

## 3. 발신 주체 · 톤

- 전부 **시스템 공지 톤**, 캐릭터 관여도 3인칭 언급("감독이 걱정하는 메시지를 보냈습니다")([03_메시지_알림](../02_기획/콘텐츠/03_메시지_알림.md) §3) — 메시지함은 캐릭터 대사 저작 불필요.
- 단 본문 서술은 1인칭 원칙 유지([00_개요](00_개요.md) §3).

## 4. 보관 · 만료

- 시즌 종료 시 인박스 **전부 비움**([03_메시지_알림](../02_기획/콘텐츠/03_메시지_알림.md) §4) — 사건은 이미 [기록 허브](03_기록.md) 로그에 있으므로 중복 보존 안 함.
- 비긴급 미확인은 페널티 없이 만료.
