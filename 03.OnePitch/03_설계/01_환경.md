## 1. 스택 결정

- **타깃**: PC(Windows/Mac/Linux) + **Android**. web 제외.
- **UI**: **Flutter (Dart)** — 가장 검증된 PC+Android 단일 코드베이스.
- **엔진**: **Rust 네이티브 라이브러리** — 게임 본체(로직·난수·시뮬·데이터·암호화).
- **경계**: **flutter_rust_bridge (FFI)** — Dart↔Rust 타입 자동 생성.
- 기존 Svelte UI·Rust 로직은 **참고 모티브로만**, 재작성.

## 2. 계층 구조

| 레이어 | 역할 | 금지 |
|---|---|---|
| **Flutter (Dart)** | 화면 렌더·입력·화면 상태·애니메이션(투구=CustomPainter/Flame) | 게임 로직, 난수 |
| **flutter_rust_bridge** | Dart↔Rust FFI 경계 (코드젠 타입드 바인딩) | — |
| **Rust 엔진 (네이티브 lib)** | 로직·난수·시뮬 + slot.db(rusqlite) + content.db 로드 + 암호화/HMAC | — |

> 계층 간 계약(누가 무엇을 하는가)은 [03_구조](03_구조.md) 참조.

## 3. 왜 이 조합인가

- **성숙도·자료 축적 최상** — Flutter 커뮤니티·패키지·SQLite 도구 방대.
- **WebView 파편화 없음** — Flutter 자체 렌더러(Skia/Impeller) → System WebView 버전 의존 소멸(Tauri/Capacitor 대비 안정성 이득).
- **Rust 네이티브 유지** — 성능 + 변조방지 옵션 보존(D4-2 은닉 밸런스). 프로토 Rust를 재구성 모티브로.
- **단일 코드베이스** — Rust는 Android `.so`(cargo-ndk) / 데스크톱 lib. Flutter가 양쪽 껍데기.

### 이 선택이 해소한 항목

| 항목 | 해소 |
|---|---|
| 인터랙티브 매치 상태 | 매치 세션을 Rust에 보유(frb 불투명 핸들) |
| 무거운 시뮬 UI 프리즈 | frb async + Rust 별도 스레드 → 논블로킹 |
| WebView 버전 파편화 | 소멸(Flutter 자체 렌더) |
| 프로토 IPC 6단계 등록비용 | frb 코드젠 자동 생성 |

## 4. 플랫폼 세부

- **저장·백업**: 위치 = PC OS 앱데이터 / Android 앱 내부저장소(권한 불필요), `path_provider` 일원화. **로컬 자동 롤링 백업만**(시즌 시작 3개 + 마이그레이션 전 백업). 수동 내보내기·클라우드 동기화 미채택(추후 가능).
- **콘텐츠 번들**: content.db 앱 번들(read-only). Android는 첫 실행·`content_version` 변경 시 내부저장소로 **추출 후 오픈**, PC는 리소스 직접 오픈 → 경로 일원화.
- **업데이트**: 앱 업데이트 = PC 스토어/인스톨러 · Android Play. 콘텐츠는 앱 업데이트에 실림. `content_version`으로 콘텐츠 OTA 미래 가능(서버 미구축). **배포 채널·수익화는 기획 비즈니스 결정으로 이월**.

## 5. 배포

- PC: Flutter desktop 번들(Windows/Mac/Linux).
- Android: `.aab`/`.apk`(Play), Rust는 cargo-ndk로 `.so`.

## 6. 빌드 · 개발 툴 (단일 Dart CLI)

프로토는 npm script 40여 개 난립(deploy·gen:manifest·build:masterdb·build:native·test:* 7종…). 새 스택은 Electron·Vite·Node·콘텐츠 파이프라인 소멸로 이미 단순 → 남은 명령을 **단일 Dart CLI**(`dart run tool <cmd>`)로 통합.

| 서브커맨드 | 역할 |
|---|---|
| `gen` | frb 코드젠(Dart↔Rust 바인딩 자동생성) |
| `run pc\|android` | 개발 실행(gen + cargo + flutter run) |
| `build pc\|android` | 릴리스 빌드(cargo/cargo-ndk + flutter build) |
| `content seed` | `data/seed/*.csv·toml`(172팀·리그·구장·특성·생성규칙 등) → content.db, 전량 재구성(`--dry-run`) — [07_데이터관리](07_데이터관리.md) §3-2 |
| `content import <file>` | AI 생성 JSON → content.db (가산, `--dry-run`) |
| `content validate` | 스키마 + 참조무결성(FK) + 번역 커버리지 검사(seed 산출물 포함) |
| `content dump` | content.db → 텍스트 스냅샷(git 이력) |
| `test` | Rust + Dart 테스트 |
| `release` | 패키징(스토어/인스톨러) |

- 단일 언어(Dart) 툴링 — 앱도 툴도 Dart, 콘텐츠 도구까지 한 곳.
- 프로토 대비 소멸: `deploy`·`gen:manifest`·`build:masterdb`·`gen:contacts`·`gen:index`·`build:packages`.
