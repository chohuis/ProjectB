## 3. 확정 순서 (의존 그래프)

일부 상수는 다른 상수 위에서만 정해진다 — 아래 순서를 지켜야 재작업이 적다.

```
1. A 능력치·성장    ← 모든 수치의 그릇(시작 밴드·XP곡선·노쇠율)
2. B 매치 판정      ← 능력치를 경기 결과로 번역(A 위에서)
3. C 평가          ← 경기 결과를 등급으로(B 위에서)
   ├ D 부상 · E 구종·훈련   ← A·B와 병행 가능
   └ F 팀 전력             ← A 집계
4. G 시장·재정 · H 평판    ← C(평가)·F(전력) 위에서
5. I 이벤트·기타           ← 나머지 참조
```

## 4. 검증 방법론 — 밸런스 시뮬 하네스

[분석서](../01_분석/프로토타입_분석서.md) §5.2 `synthetic_trajectory` 10시즌 분포 검증이 선례. 정식은 이를 **풀 커리어 시뮬**로 확장.

- **도구**: 15~20시즌 커리어를 자동으로 N회(예: 1,000회) 돌려 통계 분포를 뽑는 순수 함수형 하네스 — worldSeed만 바꿔가며, 저장 불필요·결정적.
- **통과 기준(예시)**:
  - 평균 커리어 길이 15~20시즌([01_커리어_구조](../02_기획/01_커리어_구조.md) §6)
  - S등급 희소성(남발되지 않되 에이스가 못 받는 역설도 없음 — [09_평가_시스템](../02_기획/육성코어/09_평가_시스템.md) §2 열린세부)
  - 노쇠 곡선: 파이어볼러 조기 저묾 / 기교파 지속([04_성장_곡선](../02_기획/육성코어/04_성장_곡선.md) §3)
  - 부상 빈도·최종 스탯 분포가 현실적 범위
- **워크플로**: 상수 초안 → 하네스 N회 → 분포 리포트 → 기준 이탈 시 상수 조정 → 반복. [빌드 툴](01_환경.md) §6 `dart run tool test`에 통합 후보.
