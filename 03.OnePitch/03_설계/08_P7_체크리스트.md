# P7 검증 스파이크 체크리스트

> 근거: [01_환경](01_환경.md) §3-2·§7 · [06_스키마](06_스키마.md) §6 · [00_결정_요약](00_결정_요약.md) G · 대화 설계(2026-07-14)
> 상태: **체크리스트 확정** — 실행 자체는 구현 착수 시(현재 샌드박스는 Flutter/Rust/Android 실제 툴체인 접근 불가)
> 목적: Flutter+Rust(frb)+Android+Steam은 이 프로젝트가 처음 다루는 조합. 문서·기획으로는 더 못 좁히는 "실제로 되는가"만 남은 항목을 실측으로 확정한다.

## 1. 스코프 · 원칙

- **시간박스 1~3일**. 스파이크 산출물(테스트 프로젝트)은 검증 후 **버림** — 실제 구현의 시작점이 아니라 리스크 제거용 1회성 코드.
- **목적 = "되는가" 확인이지 "잘 만드는가"가 아니다.** 최소 hello-world 수준으로만 각 항목을 통과시킨다.
- 각 항목은 **성공기준**과 **실패 시 폴백**을 미리 정해둔다 — 실측 도중 판단하지 않는다(사후 합리화 방지).
- 순서는 **의존성 순**(앞 항목이 안 되면 뒤 항목은 의미 없음) — §2.

## 2. 실측 순서

```
1. 툴체인 설치(Flutter SDK·Rust·cargo-ndk·Android SDK/NDK·frb codegen 도구)
2. frb 최소 hello-world (Dart→Rust 함수 호출, 코드젠 확인) — PC
3. rusqlite 최소 CRUD + PRAGMA(foreign_keys, journal_mode) — PC
4. PC 데스크톱 빌드 3종(Windows/Mac/Linux) — cargo build + flutter build 성공
5. content.db 번들링 + 오픈 — PC(리소스 직접 오픈)
6. Android 크로스컴파일 — cargo-ndk로 .so(arm64-v8a 최소 1종)
7. content.db 번들 → Android 내부저장소 추출 + 오픈 — path_provider
8. Android 실기기/에뮬레이터에서 frb + rusqlite 실동작
9. flutter_steamworks 연동 — PC만(Steam 클라이언트 감지 + 더미 API 호출 1개)
10. 논블로킹 확인 — 무거운 Rust 연산을 frb async로 호출 시 UI 프리즈 없는지
11. HMAC/암호화 크레이트 크로스컴파일 확인 — Android .so에도 포함되는지
```

## 3. 항목별 성공기준 · 실패 시 폴백

| # | 항목 | 성공기준 | 실패 시 폴백 |
|---|---|---|---|
| 1 | 툴체인 설치 | Flutter/Rust/cargo-ndk/Android NDK 버전 조합이 문서화된 절차로 재현 가능 | 버전 조합 재탐색, 실패해도 스택 자체를 바꾸지 않음(설치 이슈는 버전 핀으로 해결) |
| 2 | frb hello-world | Dart에서 Rust 함수 호출·반환값 수신, 코드젠 명령 1회로 성공 | frb 버전 다운/업 조정. 근본 실패 시에만 [01_환경](01_환경.md) §3-2 재검토 트리거 |
| 3 | rusqlite CRUD+PRAGMA | 테이블 생성·삽입·조회·FK 제약 위반 감지 확인 | 타입 매핑만 [06_스키마](06_스키마.md) 조정, 아키텍처 변경 아님 |
| 4 | PC 3종 빌드 | 3플랫폼 모두 실행 파일 생성+정상 기동 | 특정 OS만 실패 시 해당 OS 런처 옵션(예: Linux는 별도 배포판 이슈) 개별 조사 |
| 5 | content.db PC 오픈 | 번들 리소스에서 읽기 성공 | 경로 규약만 조정 |
| 6 | Android cargo-ndk | `.so` 생성 + APK에 정상 포함 | NDK 버전 핀 재조정. 반복 실패 시 ABI 범위 축소(우선 arm64-v8a만) |
| 7 | content.db Android 추출 | 첫 실행 시 내부저장소로 추출 후 오픈 성공 | 추출 트리거 조건(`content_version` 비교) 로직 재점검 |
| 8 | Android frb+rusqlite | 실기기/에뮬레이터에서 §2·§3과 동일 결과 | Android 전용 rusqlite 빌드 플래그(bundled feature 등) 조정 |
| 9 | flutter_steamworks | **Steamworks 정식 개발자 등록 불필요** — Valve 제공 테스트용 AppID `480`(Spacewar)을 `steam_appid.txt`에 지정, Steam 클라이언트 실행 중 상태에서 더미 stat/achievement 호출 성공 확인. 정식 등록(가입비+심사, 우리 자체 AppID)은 8단계 배포준비 시점에 별도 진행 | **DIY `steamworks-rs`로 대체** — 이때만 [01_환경](01_환경.md) §3-2 Steam 결론 재검토 (Godot 재검토 트리거는 아님, PC 배포수단 문제로 국한) |
| 10 | 논블로킹 | 수백 ms급 Rust 연산 중 Flutter UI가 계속 반응 | frb async 호출 방식(별도 isolate/thread) 조정 |
| 11 | HMAC 크로스컴파일 | 사용 크레이트가 Android NDK 타깃으로 정상 빌드 | 대체 크레이트 탐색(암호화 알고리즘 자체는 변경 없음) |

## 4. 산출물

- **버전 고정표**: Flutter·Rust·frb·cargo-ndk·Android NDK/SDK·flutter_steamworks 실측 버전 — 이후 CI·팀 재현성의 기준선.
- **[06_스키마](06_스키마.md) v1 최종 확정** — §3 항목 성공 시 DDL 후보 → 동결.
- **막힌 항목 있으면 해당 절만 재오픈** — 전면 재설계 아님(§3 폴백란이 원칙).

## 5. 스파이크 범위 밖 (착각 방지)

- 실제 게임 로직·화면 구현·콘텐츠 저작·수치 밸런싱은 여기서 안 함(전부 이후 구현 단계).
- UI 디테일(레이아웃·애니메이션)도 범위 밖 — hello-world 수준 화면 1개면 충분.
- 172팀 시드 파일 실제 생성([07_데이터관리](07_데이터관리.md) §3-3)도 범위 밖 — 스키마만 확정되면 됨.
