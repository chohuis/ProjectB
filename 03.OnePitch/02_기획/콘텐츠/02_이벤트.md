## 1. 트리거 구조 — 메커니즘 2개

| 메커니즘 | 정의 | 조건 체크 주체 |
|---|---|---|
| **폴링** | 매주 정기적으로 이벤트 시스템이 직접 조건을 평가, 맞으면 발동 | 이벤트 시스템 자신 |
| **콜백** | 다른 시스템이 자기 로직 안에서 판단을 끝내고 "지금 발동해줘"라고 직접 호출 | 호출하는 시스템(06_시장_계약·08_부상_시스템·09_평가_시스템 등) |

콜백형은 호출부가 이미 조건을 다 체크했으므로, 이벤트 시스템 쪽에 별도의 "AND 조합 엔진"이 필요 없다 — 복합조건(예: 방출경고="7월말 AND 종합지표나쁨")은 06_시장_계약 내부 로직이 처리하고, 이벤트 시스템은 결과를 받아 실행만 한다.

### 폴링의 조건 변수 — 5종 (조합 가능)

| 변수 | 예시 |
|---|---|
| 캘린더 날짜 | 올스타전, 트레이드데드라인, 드래프트, 시즌개막/종료 |
| 본인 상태 | 스탯 임계값, 관계도 임계값, 학년·연차(커리어 마일스톤), 폼(슬럼프/절정기) |
| 팀 상태 | 연패·연승 누적, 팀 특성 조건([팀_특성_시스템](../리그팀/05_팀_특성_시스템.md)) |
| 동료(캐릭터) 상태 | 동료 부상·은퇴·마일스톤 달성, 관계 라벨 임계 도달 |
| 확률 | 특별 조건 없이 가끔 발생하는 배경 이벤트 |

한 이벤트가 여러 조건 변수를 동시에 요구해도(예: "매주 체크 AND 관계도 우호 이상") 새 아키텍처가 아니라 조건절이 길어지는 것뿐이다.

### 매핑 예시

| 이벤트 | 메커니즘 | 조건 변수 |
|---|---|---|
| 올스타전 | 폴링 | 캘린더 날짜만 |
| 관계 이벤트(자연발생) | 폴링 | 동료 상태(관계도 임계) |
| 특성전용 이벤트 | 폴링 | 팀 상태(팀 특성) |
| 방출 경고 | 콜백(06_시장_계약이 판단 후 호출) | — |
| 부상 전조경고 | 콜백(08_부상_시스템이 판단 후 호출) | — |
| 갈림길A/B(진로선택) | 폴링 | 본인 상태(학년·연차) |
| 배경 플레이버 이벤트 | 폴링 | 확률만 |

## 2. 선택지 구조

- **개수 = 완전 가변**(이벤트마다 다름, 최소 2개 이상 원칙만). 고정 상한 없음 — 갈림길급 대형 이벤트(고교졸업 진로: 프로지명/대학/독립/입대)는 4갈래도 허용.
- **결과 반영 시점**: 기본은 **즉시 반영**. 단 이벤트마다 `즉시`/`지연` 플래그로 구분 가능 — [08_부상_시스템](../육성코어/08_부상_시스템.md) 치료법 선택처럼 "선택은 즉시, 효과는 지연(회복 기간 후)"인 패턴을 개별 이벤트가 가질 수 있음.

## 3. 결과·효과 범위 — 8개 대상

| 효과 대상 | 설명 |
|---|---|
| 라이브 상태 | 피로도·컨디션·사기([01_선수_능력치](../육성코어/01_선수_능력치.md) §5) — 가장 흔한 즉시효과 |
| 관계도 | [01_캐릭터](01_캐릭터.md) §5 관계 수치 증감 |
| XP/스탯 | 직접 스탯을 안 건드리고 XP를 가감 — [04_성장_곡선](../육성코어/04_성장_곡선.md) §2 XP→스탯 변환 경로를 그대로 씀. 이벤트 XP는 **특정 능력치 지정**(범용 자유 XP 아님) |
| 부상 진행 | [08_부상_시스템](../육성코어/08_부상_시스템.md) 치료 옵션 선택, 재활 진행 |
| 계약·시장 상태 | [06_시장_계약](../06_시장_계약.md) 오퍼 수락/거절, 조항 협상 |
| 주목도 | [09_평가_시스템](../육성코어/09_평가_시스템.md)·[02_아마_고교](../02_아마_고교.md) 주목도 증감 |
| 로그 기록 | [05_히스토리_엔딩](../05_히스토리_엔딩.md) 로그 6종에 항상 기록(선택 자체가 서사 흔적) |
| 후속 이벤트 해금/차단 | 선택에 따라 특정 이벤트 플래그를 켜거나 닫음(콘텐츠 분기 관리용 변수) |

## 4. 이벤트 유형 분류

별도 분류 체계를 만들지 않는다 — §1의 트리거 유형 6개(캘린더/본인상태/팀상태/동료상태/콜백/확률)가 자연스럽게 도메인 분류를 겸한다(캘린더=리그행사, 본인상태=커리어·폼, 팀상태=팀분위기, 동료상태=관계, 콜백=시스템연동, 확률=플레이버).

## 5. 빈도·쿨다운

| 트리거 유형 | 쿨다운 |
|---|---|
| 캘린더형 | 불필요 — 리그 일정이 이미 반복 주기를 정함(매년 올스타전 등) |
| 콜백형 | 불필요 — 호출하는 시스템이 발생 조건을 자체 관리(예: 방출경고 시즌당 최대 1회) |
| 본인/팀/동료 상태형 | **필요** — 같은 이벤트 재발동에 최소 간격(N주) 설정. 정확한 수치는 스탯 스케일처럼 이월 |
| 확률형 | 불필요 — 낮은 확률 자체가 자연 쿨다운 역할 |

## 7. 판별 기준 — 이벤트(02) vs 메시지·알림(03)

| | 이벤트(02) | 메시지·알림(03) |
|---|---|---|
| 기준 | 선택지가 있어 §3의 8개 효과 중 하나 이상이 실제로 갈림 | 선택지 없음(정보 전달, 많아야 확인) |
| 예시 | 부상 치료법 선택, 갈림길A/B, 계약 조건 협상 | 방출 위험 수준 통지, FA 오퍼 도착 알림(수락 여부는 별도 이벤트로 처리될 수 있음) |
