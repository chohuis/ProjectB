## 2. 의사결정 처리 방식 — 메시지는 항상 순수 통지

FA오퍼·재계약처럼 "수락/거절/카운터오퍼"가 필요한 다단계 의사결정은 메시지 안에서 처리하지 않는다.

- **메시지 = 순수 통지만.** 선택지 없음, [02_이벤트](02_이벤트.md)의 판별기준을 항상 만족.
- **실제 협상 = 전용 화면으로 위임.** 메시지를 확인하면 다단계 로직을 가진 전용 화면(협상 화면 등)으로 이동 — 이벤트(1회성 선택)로도 부적합할 만큼 복잡한 경우 이 경로를 쓴다.

## 3. 발신 주체 표시 — 시스템 공지로 통일

- 전부 **시스템 공지 톤**으로 통일. 캐릭터가 관여하는 메시지도 **3인칭 언급**만 한다 — "감독이 OO에 대해 걱정하는 메시지를 보냈습니다" 형식.
- 이로써 메시지·알림은 캐릭터 대사 저작이 전혀 필요 없다 — [01_캐릭터](01_캐릭터.md) §6 "대사 저작은 이벤트에서만" 원칙과 일치, 저작 범위가 한 번 더 좁혀짐.
