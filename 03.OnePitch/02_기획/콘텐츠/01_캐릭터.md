# 캐릭터

> 근거: [콘텐츠 개요](00_개요.md) · [01_선수_능력치](../육성코어/01_선수_능력치.md) §4(히든 스탯) · [02_스태프_능력치](../육성코어/02_스태프_능력치.md) · [팀_특성_시스템](../리그팀/05_팀_특성_시스템.md)(3슬롯 조합 패턴 재사용) · [리그팀/01_프로.md](../리그팀/01_프로.md) "주요 프랜차이즈 스타 NPC(콘텐츠)" 열린 항목을 이 문서가 해소
> 상태: **구조 확정(초안)** · 대화 설계(2026-07-13)
> 목적: 이미 존재하는 NPC(스태프·선수·기타)에 이름·성격·관계라는 서사 레이어를 얹는다.

## 1. 역할 범위 — 모든 NPC

감독·코치·구단주·팀동료·라이벌·기자/미디어·가족 등 게임에 등장하는 **모든 NPC**가 캐릭터 대상. 등급 구분(주요 vs 배경) 없이 전부 동일한 데이터 구조를 가진다.

## 2. 생성 방식 — 전원 즉시 생성 (Eager, 절차적)

- **시점**: 월드 최초 생성 시 리그팀 전체 로스터·스태프에 대해 한 번에 절차적 생성. 이후 매 시즌 신규 편입 NPC(드래프트 지명자·고교 신입생 등)도 같은 방식으로 즉시 생성.
- **완전 절차적** — 직접 저작 없음. 이름·성격 전부 규칙 기반 생성(content.db "규칙 기반 런타임 생성" 원칙, [팀_특성_시스템](../리그팀/05_팀_특성_시스템.md)과 동일 철학).
- **주인공 제외** — 콘텐츠 성격 축(§3)은 NPC 전용. 주인공은 플레이어 그 자체([00_게임_개요](../00_게임_개요.md) §5 1인칭 원칙)이므로 이 레이어를 부여하지 않는다.

## 3. 성격 — 3슬롯 조합 (팀 특성 패턴 재사용)

[팀_특성_시스템](../리그팀/05_팀_특성_시스템.md)의 3슬롯 조합 구조를 개인 단위로 재적용. 슬롯당 5개, **5×5×5 = 125가지** 조합.

### ① 관계성향 (관계수치 변화·관계 이벤트에 영향)

| 옵션 | 성격 |
|---|---|
| 사교적 | 관계 형성 빠르고 폭넓음 |
| 내성적 | 형성 느리나 한번 쌓으면 깊음 |
| 타산적 | 이해관계·성적 중심으로 관계 결정 |
| 온화함 | 갈등 회피, 완충 역할 |
| 무뚝뚝 | 표현 적어 속내 파악 어려움 |

### ② 경쟁성향 (라이벌 강도·은퇴/이적 판단·목표지향 이벤트에 영향)

| 옵션 | 성격 |
|---|---|
| 승부사 | 이기는 것 자체가 목적, 라이벌 의식 강함 |
| 완벽주의 | 자기 기준 엄격, 실패에 예민 |
| 안분 | 결과보다 과정, 압박에 덜 흔들림 |
| 야심가 | 명예·지위 추구, 상승 지향 |
| 헌신형 | 팀·타인을 위한 희생 우선 |

### ③ 감정성향 (위기 대응·서술 톤에 영향)

| 옵션 | 성격 |
|---|---|
| 침착형 | 위기에서도 냉정 유지 |
| 열정형 | 감정 기복 크나 동기부여 강함 |
| 예민형 | 압박에 취약, 슬럼프 길어질 위험 |
| 낙천형 | 실패를 빨리 털어냄 |
| 완고함 | 자기 방식 고수, 조언 안 들음 |

### 배정 방식 — 역할·소속 가중 랜덤

- 완전 균등 랜덤이 아니라 **역할**(감독·코치·기자 등)과 **소속**(팀 특성 — [팀_특성_시스템](../리그팀/05_팀_특성_시스템.md) ①철학·③위상)에 따라 슬롯 값이 쏠리게 가중.
  - 예: 스파르타(철학) 소속 선수는 "승부사"·"완벽주의" 경쟁성향이 나오기 쉬움.
  - 단, 이변 조합도 허용(예: ★★★★★ 명문 소속인데 "안분" 경쟁성향 — 팀 특성 시스템의 "이변 조합" 원칙과 동일).
- 생성 시 한 번 고정 — 이적·트레이드로 소속이 바뀌어도 성격은 유지(§5).

## 4. 인성(육성코어 히든스탯)과의 관계 — 역할 분담

[01_선수_능력치](../육성코어/01_선수_능력치.md) §4 "인성" 히든스탯과 겹쳐 보이지만 스코프가 다르다. 충돌이 아니라 별도 레이어로 병존.

| | 적용 대상 | 담당 영역 |
|---|---|---|
| **인성**(육성코어 히든스탯) | 주인공 + NPC 선수 | 육성/성장 관련(부상 재발, 훈련 스킵 등 좁은 범위) |
| **캐릭터 성격 3슬롯**(이 문서) | NPC 전용(주인공 제외), 선수·감독·코치·기자·가족 등 전 유형 | 서사/관계/NPC 판정 전반(넓은 범위) — 이벤트 발생·선택 풀, NPC 행동·판정 계수, 관계 변화 속도, 대사·서술 톤 |

## 5. 관계 수치 — 단일 축 + 라벨 버킷

- **축 개수**: 단일 축 "관계도"(신뢰도·호감도 별도로 안 쪼갬).
- **범위**: -100~100 연속치(내부), 신규 관계는 항상 **0(무난)에서 시작**.
- **표시**: 숫자 비노출, 7단계 라벨 버킷으로만 표시(FM식 "화면은 거칠게" 원칙 재사용).

| 라벨 | 성격 |
|---|---|
| 적대 | 최하위 — 공공연한 반목 |
| 불편 | 부정적, 거리 두는 관계 |
| 어색함 | 약한 부정~중립, 접점은 있으나 서먹함 |
| 무난 | 중립(기본값) |
| 우호 | 긍정적, 편안한 관계 |
| 신뢰 | 깊은 긍정, 믿고 의지 |
| 절친 | 최상위 — 각별한 유대 |

- **변화 요인**: 이벤트 선택 결과(주 요인) + 함께한 시간(같은 소속 시즌 수, 완만한 자연 상승) + 성격 궁합(§3 ①관계성향이 변화 폭·속도에 계수로 작용).
- **저장 위치**: slot.db(진행 상태) — NPC별 관계 레코드. [03_설계/02_데이터](../../03_설계/02_데이터.md) 3계층 원칙과 일치.
- **정확한 구간 경계값**: 스탯 스케일 확정 후(다른 시스템과 동일 패턴).

## 6. 성격이 영향을 주는 지점 (효과 매핑)

멘트/대사 뱅크는 별도로 만들지 않는다 — 필요한 대사는 [02_이벤트](02_이벤트.md) 저작에 녹인다(대사가 실제로 쓰이는 유일한 콘텐츠 카테고리). 성격의 실질 역할은 아래 4곳의 **계수·태그**다.

| 영향 지점 | 방식 |
|---|---|
| 이벤트 발생·선택 풀 | 어떤 이벤트가 뜨는지, 어떤 빈도로 발생하는지의 트리거 조건에 성격 태그가 가중치로 반영 |
| 관계 수치 변화 속도·폭 | §5 변화 요인의 성격 궁합 계수 |
| NPC 행동·판정 | 감독 역할배정 방식, 라이벌 도발 빈도, 구단주 인내심([06_시장_계약](../06_시장_계약.md) §1) 등 시스템적 의사결정 계수 |
| 대사·서술 톤 | [06_서술_템플릿](06_서술_템플릿.md)에서 인용구 삽입 시 워딩 톤 참조(직접 대사 저작이 아니라 톤 지시자로만 관여) |

## 7. 수명주기 — 은퇴·이적·방출 연동

| 이벤트 | 캐릭터 데이터 처리 |
|---|---|
| **은퇴** | 활동 풀에서 제외(더 이상 등장 안 함). 관계 레코드·성격 데이터는 **영구 보존**([05_히스토리_엔딩](../05_히스토리_엔딩.md) 관계로그가 과거 인연을 계속 참조하므로 삭제 불가) |
| **이적/트레이드** | 소속만 변경. 성격 슬롯(생성 시 고정)과 관계도는 그대로 유지 |
| **방출** | [06_시장_계약](../06_시장_계약.md) §1과 연동. 캐릭터 데이터 자체는 유지(재계약·독립리그 재도전 등 후속 경로에서 계속 등장 가능) |
| **신규 편입** | 매 시즌 드래프트·신입생 등 새 NPC 등장 시 §2 Eager 생성 방식 동일 적용 |
