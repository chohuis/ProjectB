## 1. 두 개의 독립된 축

| 축 | 무엇 | 적용 범위 |
|---|---|---|
| **축1 — 경기 유형(관여도)** | 이 경기가 주인공과 얼마나 관련 있는가 | 모든 경기 |
| **축2 — 진행 모드(개입 수준)** | 주인공이 등판할 때 플레이어가 얼마나 직접 조작할지 | 주인공 등판 경기에만 |

두 축은 **독립적**이다 — 축1이 "이 경기를 왜 보여주는가"를 정하고, 축2는 "주인공이 던질 때 얼마나 손대는가"를 정한다.

## 2. 축1 — 경기 유형(관여도)

| 유형 | 설명 | 기본 표시 |
|---|---|---|
| **A. 주인공 등판 경기** | 주인공이 그 경기에서 던짐(선발 또는 구원 등판) | §3(축2 모드)에 따름 |
| **B. 주인공 소속팀 미등판 경기** | 주인공 팀 경기지만 주인공은 안 던짐 | 이닝별 요약 텍스트 |
| **C. 주인공 무관 경기** | 다른 팀들끼리 경기 | 순수 배경 — 결과·스탯만 산출, 텍스트 최소 |

- **선발 vs 구원 등판**: 구원으로 등판하는 경우, 등판 전 이닝은 **유형 B처럼** 처리되다가 실제 등판 시점부터 유형 A(§3 모드 적용)로 전환.

## 3. 축2 — 진행 모드 (주인공 등판 경기 전용, 경기 시작 전 1회 선택·고정)

| 모드 | 설명 |
|---|---|
| **자동** | 전체 경기 자동 진행. 결정적 순간도 AI가 대신 구종·코스 선택. 플레이어는 결과만 관전(요약 텍스트) |
| **수동** | 전체 경기 **1구 1구** 플레이어가 직접 구종·코스 선택 |
| **반자동** | 평상시(이닝요약/타석한줄) 자동 진행 + **결정적 순간만** 1구 단위 조작 요청 |

- **선택 시점**: 경기 시작 전 1회, 그 경기 내내 고정(경기 중 전환 불가).
- **반자동의 "결정적 순간" 트리거**: 위기상황·개인기록근접·라이벌매치업(복수 발생 가능, 이닝 중 동적 격상) — 세부는 §4.

## 4. 반자동 모드의 격상 트리거

| 트리거 | 내용 |
|---|---|
| 위기상황 | 만루·동점·역전 기회 등 레버리지 높은 타석 |
| 개인기록 근접 | 노히트노런·퍼펙트·탈삼진 신기록 등 임박 |
| 라이벌 매치업 | 특정 라이벌 NPC 타자와의 대결 |

- 사전 스크립트가 아니라 **이닝 진행 중 실시간 동적 격상**(예: 2사 후 만루가 되면 그 타석이 즉시 격상).

## 5. 1구 조작 — 진짜 볼카운트 시뮬레이션

한 타석 = **여러 번의 1구 조작이 누적**되는 진짜 야구 카운트 시뮬(추상화된 단발 판정 아님).

- **구종 선택**: 보유 구종([05_구종_시스템](05_구종_시스템.md) 카탈로그 중 습득한 것만).
- **코스 선택**: 스트라이크존 **3×3 그리드(9분할)**.
- **카운트 진행**: 0-0에서 시작 → 매 구마다 볼/스트라이크/파울/인플레이/**사구** 판정 → 카운트 누적(예: 1-0→1-1→2-1...) → **3스트라이크=삼진, 4볼=볼넷, 사구=즉시 1루 진루(볼넷과 동일 처리), 인플레이 발생 시 타석 종료**(§6 결과 확장으로).
- **사구 확률**: 제구 낮을수록↑, 코스가 3×3 그리드의 몸쪽(안쪽 열)일수록↑.
- 판정 = 선택 구종 마스터리 단계 + 관련 능력치(제구·구위) vs 타자 능력치(컨택·선구안·파워) + 상황 보정(피로도·컨디션·클러치) — **정확한 판정식은 스탯 스케일 확정 후**.

## 6. 타석 결과 확장 (인플레이 이후)

인플레이(공을 맞힘) 발생 시, 단순 "안타/아웃"이 아니라 아래로 세분화된다.

| 결과 | 판정 근거 |
|---|---|
| 단타·2루타·3루타·홈런 | 타자 파워 + 수비 위치 + 상황 확률로 세분화 |
| 아웃(플라이·땅볼) | 기본 아웃 |
| **병살타** | 인플레이 타구 + 주자 상황(1루 주자 등) 기반 자동 판정 — 투수의 능동 선택 아님, 시스템 처리 |
| **희생플라이·희생번트** | 주자 상황 + 아웃카운트 기반 자동 판정 |
| **실책** | [03_팀_전력_공식](03_팀_전력_공식.md) "수비 전력" 확률로 발생 — 발생 시 **비자책점**으로 별도 집계(§9) |

## 6-1. 수비 시프트

- 타자의 [타격 유형 태그](05_구종_시스템.md) §5(파워형·컨택형·스프레이형 등)에 따라 수비 시프트가 자동 적용된다.
- **파워형(당겨치기 성향)** → 시프트 강하게 적용, 해당 방향 타구 아웃 확률↑(안타 확률↓) / **스프레이형** → 시프트 효과 약함(전방향 타격이라 시프트 무력화).
- 감독 [02_스태프_능력치](02_스태프_능력치.md) 전술력이 시프트 적중도에 소폭 영향.

## 7. DH(지명타자) 제도 — 적용

- **전 리그 DH 적용** — 투수(주인공 포함)는 **타석에 서지 않는다**.
- 함의: 주인공 전용 타격 조작 시스템 불필요 — 게임 범위가 "투수로서 던지는 것"에 완전히 집중됨.

## 8. 감독 개입 — 투수 교체 타이밍

| 모드 | 처리 |
|---|---|
| 자동·반자동 | **감독 AI**가 [02_스태프_능력치](02_스태프_능력치.md) 전술력 + 신뢰형성력 + 주인공 [01_선수_능력치](01_선수_능력치.md) 피로도(라이브상태) + 투구수 규정을 종합해 교체 시점 판단 |
| 수동 | 이닝 종료·위기상황마다 플레이어에게 **"교체할까요?"** 판단 기회 제공. 선택 안 하면 자동판단이 디폴트로 개입 |

- 이 판단(특히 플레이어가 감독 의견에 반해 강행하거나 순순히 따르는 것)이 [04_프로_커리어](../04_프로_커리어.md) Phase3 "감독 신뢰" 형성에 영향.

## 9. 주루 플레이 — 도루

- **판정**: 타자(주자) "스피드" vs 투수 "경기운영"(견제 능력을 여기 흡수 — 별도 스탯 신설 안 함, §10) + 포수 "수비" 스탯.
- 도루 시도·성공·실패(주루사)가 인플레이 사이에 발생 가능한 별도 이벤트로 처리.

## 10. 견제 — 별도 스탯 없음

- 견제(주자 견제·도루 저지 기여)는 **투수 "경기운영" 스탯에 흡수**됨 — 9개 능력치 체계를 유지하기 위해 별도 스탯을 신설하지 않음.
- 경기운영이 높은 투수 = 이닝 소화 효율뿐 아니라 주자 견제도 잘함(같은 "영리하게 경기를 푸는 능력"으로 통합 해석).

## 10-1. 환경 요소 — 날씨 · 파크팩터

- **날씨**: 경기마다 결정되는 조건(맑음·흐림·비·강풍 등)이 소폭 모디파이어로 작용 — 비=제구 하락 확률↑, 강풍=장타(홈런) 확률 변동, 더위=체력 소모↑([01_선수_능력치](01_선수_능력치.md) 라이브상태 피로도와 연동).
- **파크팩터**: 구장마다 성격이 다름(좁은구장=홈런 잘 나옴 / 넓은구장=홈런 적고 수비범위 중요 / 투수친화구장 등). 인플레이 결과(§6) 확률에 구장 성격이 가중된다.
- **구장 정보 소속**: 파크팩터가 존재하려면 **구단(팀)에 구장 데이터가 있어야 함**:
  - **프로**: 팀별 전용 홈구장 — [01_프로](../리그팀/01_프로.md) §9 "팀별 홈구장·연고 도시 상세"(기존 열린세부)를 **파크팩터 성격까지 포함하도록 확장** 필요.
  - **대학**: 이미 [03_대학](../리그팀/03_대학.md)에 **거점구장 방식**이 있음 — 거점구장 1곳당 파크팩터 1개, 여러 팀이 공유.
  - **고교·독립**: 구장 데이터 미정 — 고교는 권역별 홈경기장, 독립은 팀별 구장 필요 여부 추후 결정.

## 11. 핵심 원칙 — 시뮬레이션 해상도와 표시 해상도는 별개

**어떤 축1 유형·축2 모드로 진행하든, 내부 시뮬레이션은 항상 최소 타석(PA) 단위로 풀 연산된다.** 리그팀 문서에서 이미 확정한 "전 리그 전부 풀 시뮬" 원칙과 일치.

- **자동 모드**에서도 실제로는 매 타석 AI가 구종·코스를 대신 선택해 동일한 판정 로직을 거친다 — 단지 **화면에 안 보여주고 안 물어볼 뿐**.
- 축1 유형 C(주인공 무관 경기)도 마찬가지로 풀 연산 — 순위표·라이벌 추적·기록의 근거가 됨.
- → **모드·유형은 "표시·개입 수준"만 바꾸고, 기록·스탯의 정확도는 절대 바뀌지 않는다.**

## 12. 결과물(스탯·기록) 관리

- **생성 데이터**: 이닝별 라인 → 타석별 결과(볼카운트 진행 포함) → 선수별(투수·타자 전원) 누적 스탯.
- **자책점/비자책점 구분**: 실책(§6) 개입으로 실점 시 **비자책점**으로 별도 집계 — ERA 등 투수 기록의 정확성에 필수.
- **저장 경로** — [데이터 아키텍처](../../03_설계/02_데이터.md) slot.db 5그룹과 연동:
  - 시즌 중: `season_stats`에 주 단위 누적(휘발, 시즌 한정).
  - 시즌 종료: 주인공은 `career_history`+`game_log`에 **경기별 전체 보존**(압축 안 함 — 히스토리 엔딩의 본체). Named NPC는 연도 라인으로 압축. 배경 NPC는 연도 라인만.
- **모드·유형 무관 동일 기록**: 자동으로 돌리든 수동으로 직접 던지든, 경기 결과·개인 기록은 **동일하게 정확히 남는다** — 어떤 모드를 쓰든 커리어 기록상 불이익 없음(공정성).
- **기록 필드**(구체 목록, 상세는 콘텐츠/구현 단계):
  - 투수: 승·패·세이브·홀드·ERA·이닝·탈삼진·볼넷·피안타·피홈런·WHIP 등
  - 타자: 타율·출루율·장타율·OPS·홈런·타점·득점·도루 등
