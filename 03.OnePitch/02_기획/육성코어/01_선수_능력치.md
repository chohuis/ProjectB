# 선수 능력치 체계 (투수 · 타자)

> 근거: [육성코어 개요](00_개요.md) · 대화 설계(2026-07-09)
> 상태: **확정(초안) — 수치 스케일까지 확정 완료(§7)** · 프로토(velocity/command/control/movement/mentality/stamina/recovery/clutch/holdRunners 9종)는 참고하지 않음 — 완전 백지 재설계.

## 1. 설계 원칙

- **3카테고리 대칭 구조**(피지컬·기술·멘탈) — [팀 특성 시스템](../리그팀/05_팀_특성_시스템.md)의 3슬롯 언어와도 통일.
- 각 카테고리 3개씩, **총 9개 노출 능력치**.
- **투수·타자 모두 15개**(9노출 + 히든3 + 라이브3) — 매치 엔진에서 투수 vs 타자가 같은 해상도로 맞붙도록 대칭 설계.
- 능력치는 이미 만든 시스템과 최대한 맞물리게: 회복력↔투구수 휴식표, 경기운영↔에이스 딜레마.

## 2. 투수 능력치 (9 노출 + 3 히든)

| 카테고리 | 능력치 | 무엇 |
|---|---|---|
| **피지컬** | 구속 | 순수 구속(구위의 원재료) |
| | 체력 | 경기 내 이닝 소화 지구력 |
| | 회복력 | 등판 간 회복 속도 — 투구수 휴식표와 직결 |
| **기술** | 제구 | 커맨드·로케이션 정밀도 |
| | 구위(무브먼트) | 공의 움직임·위력 (구종별 개별 마스터리는 별도 [구종 시스템](05_구종_시스템.md)에서 — 여기와 안 겹침) |
| | 경기 운영 | 적은 투구수로 아웃 잡는 능력 — 에이스 딜레마(밀어붙이기 vs 아끼기)의 실제 변수. **주자 견제·도루 저지 기여도 흡수**([07_매치_엔진](07_매치_엔진.md) §10) |
| **멘탈** | 클러치 | 위기 상황 퍼포먼스 |
| | 침착함 | 꾸준함·빅이닝 방지 |
| | 리더십 | 팀 내 영향력·서사(개인 성적보다 관계·이벤트에 작용) |

## 3. 타자 능력치 (9 노출 + 3 히든, 투수와 대칭)

| 카테고리 | 능력치 | 무엇 |
|---|---|---|
| **피지컬** | 파워 | 장타력·홈런 생산력 |
| | 스피드 | 주력·주루 |
| | 체력 | 풀시즌 내구성(투수는 이닝단위, 타자는 시즌단위 부상저항) |
| **기술** | 컨택 | 정확도·삼진 회피 |
| | 선구안 | 볼넷 유도·유인구 대응 |
| | 수비 | 포지션 수비력 — 투수에 없는 타자만의 고유 축 |
| **멘탈** | 클러치 | 위기 상황 타격 (투수와 동일 개념) |
| | 침착함 | 슬럼프 저항·꾸준함 |
| | 리더십 | 팀 서사·관계 (투수와 동일 개념) |

### 대칭 원리

- **피지컬·멘탈**은 투수↔타자 개념이 거의 그대로 대응(구속↔파워, 회복력↔체력, 클러치·침착함·리더십은 동일).
- **기술만 완전히 다른 축**(제구/구위/경기운영 ↔ 컨택/선구안/수비) — 투수는 "던지는 기술", 타자는 "맞히고 지키는 기술"이라 자연 분화.

## 4. 히든 스탯 (3종, 투수·타자 공통, UI 비노출)

| 히든 스탯 | 의미 |
|---|---|
| **천재성** | 타고난 잠재력·성장 상한(재능 캡). 훈련 효율을 은밀히 좌우 |
| **인성** | 팀워크·사생활 리스크·이벤트 성향(사고뭉치 vs 모범생 등 서사 분기) |
| **성실함** | 훈련 꾸준함·자기관리(방심·부상 재발 등에 영향) |

- **UI 비노출**: 수치를 직접 보여주지 않음. 플레이어는 **행동·성장 패턴을 관찰**해서 짐작해야 함(FM 스카우팅 미스터리 감각).
- **드러나는 방식**: 성장 속도가 기대보다 빠르거나 느림(천재성 암시), 특정 이벤트 발생 빈도·선택지 색깔(인성), 훈련 스킵·부상 재발 패턴(성실함) 등으로 **간접 노출**.
- 주인공에게도 적용(자기 자신의 히든 스탯도 완전히는 모름 — 코치 피드백·이벤트로 짐작) vs NPC 전용(스카우팅 대상) — **적용 범위는 §7 열린 세부**.

## 5. 라이브 상태 (3종, 영구 스탯과 별개)

12개 능력치는 전부 **영구 성장형**이다. 여기에 매주 변동하는 **일시적 상태값**을 별도로 둔다 — 훈련·매치엔진·부상 시스템 셋 다에서 소비되므로, 파편화를 막기 위해 "선수 데이터의 단일 출처"인 이 문서에 정의하고 [06_훈련_시스템](06_훈련_시스템.md)·[07_매치_엔진](07_매치_엔진.md)·[08_부상_시스템](08_부상_시스템.md)은 여기를 참조한다.

| 라이브 상태 | 무엇 | 변동 |
|---|---|---|
| **피로도** | 지금 당장 얼마나 지쳐있는가(체력=그릇 크기, 피로도=현재 소모량) | 등판·훈련으로 축적, 휴식으로 감소 — 투구수 휴식표와 직결 |
| **컨디션(폼)** | 단기 상승세·슬럼프 | 경기 결과·연속성으로 변동, 퍼포먼스에 순간 가감(+/-) |
| **사기** | 팀 관계·이벤트 반응 | 팀 성적·이벤트·역할 변동으로 오르내림 |

- **저장 위치**: slot.db(파생 상태 아님 — 실제 진행 상태라 매주 갱신·영속). 영구 능력치와 스키마상 구분.

## 6. 총계

- 투수 = 9(노출) + 3(히든) + 3(라이브) = **15개**
- 타자 = 9(노출) + 3(히든) + 3(라이브) = **15개**

## 7. 수치 스케일 (확정 — 전 시스템 공통 표준)

대화 설계(2026-07-09). Football Manager식 "화면은 거칠게, 내부는 정밀하게" 하이브리드 패턴을 채택하되, 실제 야구 스카우팅 관행(20~80)을 유지해 [00_게임_개요](../00_게임_개요.md) "진지한 사실주의" 톤을 지킨다.

| 항목 | 값 |
|---|---|
| **범위** | **20~80**(실제 MLB 스카우팅 등급 관행 — 50=평균, 80=최상급) |
| **내부 정밀도** | 연속치(소수점 단위) — 훈련·경기마다 미세하게 누적, 절대 반올림 없이 추적 |
| **화면 표시** | **3단위 반올림**(20·23·26···80 = 21버킷) — 5단위(13버킷, 진짜 스카우팅)보다 촘촘해 FM(20버킷)급 성장 체감 빈도 확보, 동시에 "68등급 어깨" 같은 스카우팅 언어는 유지 |
| **적용 범위** | 선수 능력치([01_선수_능력치](01_선수_능력치.md) 전체) + [02_스태프_능력치](02_스태프_능력치.md) + 기타 전 수치 시스템의 **공통 표준** |

- **왜 이 조합인가**: 순수 0~100은 "68과 65가 실제로 다른가?"라는 허위정밀도 문제, 순수 20~80(5단위)은 장기 성장곡선(15~20시즌)을 담기엔 거침, 순수 1~5는 성장 서사를 담을 그릇 자체가 없음. 하이브리드가 셋의 단점을 모두 피함.
- 이 표준이 확정되면서 각 육성코어 문서·[06_시장_계약](../06_시장_계약.md)·[리그팀](../리그팀/00_개요.md)에 이월돼 있던 "정확한 수치" 항목들을 구체화할 수 있게 됨(다음 단계).
