# 팀 전력 공식

> 근거: [육성코어 개요](00_개요.md) · [01_선수_능력치](01_선수_능력치.md) · [02_스태프_능력치](02_스태프_능력치.md) · [팀 특성 시스템](../리그팀/05_팀_특성_시스템.md)
> 상태: **구조 확정(초안)** · 정확한 가중치 수치는 스탯 스케일 확정 후

## 1. 원칙 — 두 가지 "전력"을 구분한다

| 구분 | 정체 | 저장 여부 |
|---|---|---|
| **전력★(1~5)** | 리그팀 문서에 이미 매긴 값 — **설계 타깃**. 선수 생성 규칙([content.db](../../03_설계/02_데이터.md) generation_rules)이 이 값을 향해 로스터를 생성 | 콘텐츠(정의) — content.db |
| **실전력** | 현재 로스터+스태프+팀특성에서 **매번 계산**되는 값. 선수가 성장·트레이드·졸업하면 자연히 변동 | **파생(캐시)** — 저장 안 함, slot.db에서 매번 재계산 |

- 이 문서는 **실전력을 산출하는 공식**을 정의한다. 새 게임 시작 시점엔 실전력이 전력★ 타깃에 가깝게 맞춰지도록 선수 생성 규칙이 조정되고, 이후엔 자연스럽게 갈라진다(육성 성공/실패, 부상, 세대교체).

## 2. 공식 구조

```
팀 전력 = f(선발진, 불펜, 타선, 수비) × 스태프 보정 × 팀특성 보정
```

## 3. 4개 하위 지표

매치 엔진에서 투수진 vs 타선의 실제 맞대결에 개별로 쓰인다.

| 하위지표 | 산출 근거 |
|---|---|
| **선발진 전력** | 선발 투수진의 구속·제구·구위·경기운영 가중합 |
| **불펜 전력** | 구원 투수진의 동일 지표(체력·회복력 가중 ↑ — 짧은 등판 특성 반영) |
| **타선 전력** | 야수진의 파워·컨택·선구안 가중합 |
| **수비 전력** | 야수진의 수비 스탯 가중합 |

## 4. 스태프 보정

```
스태프 보정 = f(감독 전술력, 코치 종합지도력 평균, 코치 적성매칭도)
```

- **코치 적성매칭도**: 보직에 적성대로 배치된 비율 — [02_스태프_능력치](02_스태프_능력치.md) §2-3의 미스매치 페널티가 여기서 실제 수치로 반영됨.
- 배수(곱)로 적용 — 좋은 스태프는 같은 로스터도 더 잘 활용.

## 5. 팀특성 보정

[팀 특성 시스템](../리그팀/05_팀_특성_시스템.md) §2의 효과 매핑을 실제 공식에 연결:

| 특성 슬롯 | 하위지표 보정 |
|---|---|
| ① 철학 = 공격야구 | 타선 전력 ↑ |
| ① 철학 = 수비/짜임새 | 수비 전력 ↑ |
| ① 철학 = 투수왕국 | 선발진 전력 ↑ |
| ② 자원 = 부유 | 전 지표 소폭 ↑ (시설 보너스) |
| ② 자원 = 궁핍 | 전 지표 소폭 ↓ |

## 6. ★ 환산

- 종합 전력치(4지표×스태프×특성 합산 결과)를 **1~5★ 구간으로 매핑**해 UI·리그팀 문서의 기존 전력★ 표기와 그대로 연결.
- 시즌 중 실전력이 표기된 전력★와 벌어지면(선수 성장·부상·트레이드) — 뉴스·순위 예측 이변의 재료가 됨("전력★2인데 3위를 달리는 이변의 팀" 같은 서사).
