# 히스토리 · 엔딩 (Phase 6)

> 근거: [게임 개요](00_게임_개요.md)(원 기획 계승 철학 "엔딩은 판정이 아니라 기록" — 아래 §배경) · [커리어 구조](01_커리어_구조.md) Phase 6 · [07_매치_엔진](육성코어/07_매치_엔진.md) §12(career_history 저장) · [09_평가_시스템](육성코어/09_평가_시스템.md)

> 원 정체성의 본체 — 프로토타입에서 가장 미완성이던 영역, 이번 정식버전 최우선 신규 개발.

## 배경 (원 기획 계승 철학)

- "이 게임의 엔딩은 판정이 아니라 플레이 내/외부에서 쌓인 로그가 뒤를 받쳐 주어야 한다."
- "경기별 핵심 포인트가 빠지면 히스토리 전체가 무의미해진다."
- 데이터 저장 자체는 이미 [07_매치_엔진](육성코어/07_매치_엔진.md) §12에서 확정: 주인공은 career_history·game_log에 **경기별 전체 보존**(압축 안 함).

## 1. 범위 — 상시 기록보관소 + 은퇴 화면

- **상시 기록보관소**: 플레이 중 언제든 "기록" 탭을 열어 지난 경기·사건·수상을 돌아볼 수 있음(원형의 "기록 아카이브" 계승). **업적**([콘텐츠/04_업적](콘텐츠/04_업적.md), 세계관 밖 메타 체크리스트)도 이 화면 안의 별도 탭으로 표시 — 별도 화면 신설 안 함.
- **은퇴 화면**: 은퇴 시점에 전체를 응축하는 특별한 엔딩 연출이 별도로 있음(§4).

## 4. 은퇴 화면 — 하이브리드(서술+통계)

핵심 원칙: **"인생은 점수로 판정하지 않는다"** — 등급·점수 매김 없이, 기록과 이야기로 응축한다.

| 요소 | 내용 |
|---|---|
| **서술형 커리어 내레이션** | 주요 시즌·사건·라이벌 관계를 엮은 이야기 텍스트(템플릿 기반 생성) |
| **통산 기록 대시보드** | ERA·승패·탈삼진 등 숫자, 등급 매김 없이 순수 기록으로 + **최종 자산**([08_개인_재정](08_개인_재정.md) §4, 등수 아닌 순수 기록) |
| **커리어 타임라인 그래프** | 시즌별 성적 곡선 시각화(무명→정상→황혼의 곡선이 한눈에) |
| **하이라이트 모음** | 가장 기억에 남는 경기 몇 개 재조명(개인기록·라이벌전 등) |
| **은퇴식 연출** | 팬·동료·감독의 반응(컷신 텍스트) |
| **명예 요소** | 수상 목록·타이틀 히스토리 |

## 4-1. 은퇴 이후 — 회차 진행 (완전 독립, 대화 설계 2026-07-14)

- 은퇴 화면 확인 후 **메인 메뉴로 복귀** → "새로 시작"은 완전히 새로운 세이브 슬롯(새 worldSeed)을 생성 — [03_설계/02_데이터](../03_설계/02_데이터.md) "파일=슬롯" 원칙이 이미 지원하는 구조라 **새 메커니즘 불필요**.
- **회차 간 연결 없음**: 이전에 은퇴한 주인공의 기록·업적·유산이 다음 플레이에 전혀 영향을 주지 않음 — 명예의 전당·글로벌 프로필 같은 세이브 밖 저장소를 만들지 않는다(스코프 의도적 배제).
- 기존 세이브 슬롯은 은퇴 후에도 그대로 남아있어([05_히스토리_엔딩] 기록보관소·은퇴 화면 다시 열람 가능), 플레이어가 여러 슬롯을 병행해 여러 "인생"을 독립적으로 보관할 수 있다 — 이 정도가 이 게임이 지원하는 회차 진행의 전부.
