## 4. 문서 관리 (리그별 통합)

각 리그 = **팀 + 리그·대회 운영을 한 파일**로 통합. 공통(권역·분포·시뮬)만 이 개요에.
```
02_기획/리그팀/
├── 00_개요.md   ← 프레임워크(8권역·분포·시뮬 범위)  [공통]
├── 01_프로.md   ← 프로 팀(1군10+2군10) + 운영 (144경기+5강PO/99경기+축약PO, 전부 풀 시뮬)
├── 02_고교.md   ← 고교 팀(102) + 리그·대회 운영
├── 03_대학.md   ← 대학 팀(50) + 운영
├── 04_독립.md   ← 독립 팀(10) + 운영
└── 05_팀_특성_시스템.md   ← 3슬롯 특성 조합(전 팀 선택 가능화) + 전력★ 가중 배치
```

## 4-1. 시뮬 범위 (모든 리그 공통) — 하이브리드

182팀(고교102+대학50+독립10+프로1군10+2군10)을 전부 실제 경기시키지 않는 것이 원칙이나, **모든 리그가 예외적으로 전부 풀 시뮬로 확정**됐다(각 리그팀 문서 참조). 아래 표는 향후 세계가 커질 경우(해외리그 DLC 등)를 위한 기본 정책으로 유지.

| 반경 | 대상 | 처리 |
|---|---|---|
| **풀** | 주인공 소속 리그/권역 + 참가 대회 | 실제 경기 시뮬 |
| **간이** | 인접 리그(상위 무대 등) | 전력 기반 결과·순위만 |
| **결과만** | 먼 리그/권역 | 순위표 산출(라이벌·연감용) |

- 기술 설계의 반경 시뮬과 연동. 주인공 스테이지 이동 시 활성 범위도 이동.
- **현재 전 리그가 예외 적용**: 고교·대학·독립·프로(1군720+2군495경기 포함) 전부 **풀 시뮬 확정** — 세계 규모가 아직 국내로 한정돼 있어 일관성을 우선. 해외 리그(DLC) 추가 시 하이브리드 원칙 재적용 검토.

## 5. 팀 명명 정책 (확정)

- 가상 세계(실명 아님) → 팀명·연고 전부 **가공**, 실제 한국 야구 조직·대회명 인용 금지(문서 작성 규칙).
- **전부 기획자(AI)가 제안**하고 사용자가 검토·조정하는 방식으로 확정. 핵심 팀(선택 가능 팀)은 상세 명명 완료, 배경 팀도 전 리그 명명 완료(각 리그팀 문서 참조).
