# Match Engine Simulator Guide

## 1) 목적
- `Ctrl+Q` 개발자 도구의 **매치 엔진 시뮬레이터**는 매치 엔진 밸런스 수치를 조정하고 즉시 검증/저장하는 관리자 기능이다.
- 저장 시 `resource/data/master/balance/match_engine_tuning.json`에 반영되며, Debug/Release 공통 설정으로 사용된다.

## 2) 진입 방법
1. 메인 화면에서 `Ctrl+Q`
2. `매치 엔진 시뮬레이터` 선택

## 3) 버튼 동작
- `다시 불러오기`: 현재 파일 기준 튜닝값 재로드
- `세션 즉시 반영`: 파일 저장 없이 현재 실행 세션 엔진에 반영
- `저장 (실환경 반영)`: 파일에 영구 저장 + 엔진 반영
- `Gate 실패 시 강제 저장`: 스모크 게이트 실패를 우회해 저장 (스키마 오류는 우회 불가)

## 4) 저장 전 자동 검증
- 스키마 검증: `match_engine_tuning.schema.json` 기준으로 필수 키/범위 검증
- 스모크 시뮬: 저장 직전 기본 20게임 자동 실행
- 게이트 실패 시 기본 동작: 저장 차단

## 5) 시뮬레이션 결과 확인 방법
시뮬레이터 하단 `시뮬레이션` 카드에서 확인한다.

- `Games`: 실행 게임 수
- `Avg Score (Away/Home/Total)`: 평균 원정/홈/총 득점
- `Avg Pitches (P50/P90)`:
  - `Avg`: 경기당 평균 투구 이벤트 수
  - `P50`: 중앙값(보통 경기 길이)
  - `P90`: 상위 10% 장기전 길이
- `BB% / K% / HR%`:
  - `BB%`: 볼넷 비율
  - `K%`: 스트라이크 결과(루킹/헛스윙) 비율
  - `HR%`: 홈런 비율
- `Result Rates(%)`: 결과 코드별 전체 비율

## 6) Gate Range 해석
- 화면에 `Gate Range`가 같이 표시된다.
- 각 KPI가 범위를 벗어나면 `Smoke Gate 실패` 목록에 항목이 표시된다.
- 실패 상태에서 저장하려면 `Gate 실패 시 강제 저장` 체크 후 저장한다.

## 7) 권장 운영 순서
1. 수치 조정
2. `세션 즉시 반영`
3. `배치 시뮬 실행`으로 지표 확인
4. Gate 통과 확인 후 `저장 (실환경 반영)`

## 8) 문제 발생 시
- 헤더만 보이고 본문이 안 뜨면 앱 재시작 후 재시도
- `다시 불러오기` 실패 시 에러 메시지 확인 후 스키마/JSON 키 누락 여부 점검
