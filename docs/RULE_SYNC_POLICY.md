# Rule Sync Policy

## 목적
- 규칙 문서 변경 사항이 Git pull만으로 모든 개발 PC에 동일하게 적용되도록 보장한다.

## 단일 소스 원칙
- 아래 문서를 규칙의 공식 소스로 사용한다.
- `docs/DEVELOPMENT_RULES.md`
- `docs/GIT_WORKFLOW.md`
- `docs/ENCODING_POLICY.md`
- `docs/RULE_SYNC_POLICY.md`

## 변경 절차
1. 규칙 변경 필요 사항을 문서에 먼저 반영한다.
2. 문서 변경 이유를 커밋 메시지에 명확히 기록한다.
3. 규칙 변경 커밋 이후에 구현/코드 변경을 진행한다.

## 작업 시작 체크리스트 (모든 PC 공통)
1. `git pull`
2. 규칙 문서 4개 최신 내용 확인
3. 현재 브랜치에서 규칙 문서 로컬 변경 여부 확인

## 충돌 처리
- 규칙 문서 충돌이 발생하면 충돌 해결 전 기능 개발을 진행하지 않는다.
- 충돌 해결 후 팀/사용자에게 변경 요약을 공유한다.
